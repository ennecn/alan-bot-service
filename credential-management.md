# Credential Management System

## Overview

Centralized credential store on Mac Mini, shared by all 4 OpenClaw bots. Credentials are stored in a JSON file, exported as environment variables, and mounted into Docker containers.

## Architecture

```
cred-manager.sh (Mac Mini host)
  ├─ store.json (/Users/fangjin/.credentials/store.json)
  │    └─ JSON: {credentials: {NAME: {value, description, added_at}}}
  ├─ env.secrets (/Users/fangjin/.credentials/env.secrets)
  │    └─ KEY=VALUE format, auto-generated by `export`
  └─ NAS backup (/Users/fangjin/nas/.credentials/)
       └─ store-YYYYMMDD-HHMMSS.json + store-latest.json

Docker containers (all 4 bots):
  ├─ env_file: /Users/fangjin/.credentials/env.secrets  (loaded at start)
  └─ volume: env.secrets → /mnt/credentials/env.secrets:ro  (runtime access)
```

## Management Script

Location: `/Users/fangjin/cred-manager.sh`

| Command | Description |
|---------|-------------|
| `add <NAME> <VALUE> [DESC]` | Add a credential |
| `list` | List all credentials (name + description) |
| `get <NAME>` | Get credential value |
| `delete <NAME>` | Remove a credential |
| `export` | Generate env.secrets from store |
| `backup` | Backup store.json to NAS |
| `restore [FILE]` | Restore from NAS backup |
| `import <FILE>` | Bulk import from .env file |

## How Bots Use Credentials

### Reading (inside container)
```bash
# Source latest credentials (works even for newly added ones)
source /mnt/credentials/env.secrets
echo $CLOUDFLARE_API_TOKEN
```

### Adding (via nodes tool)
```json
{
  "action": "run",
  "node": "MacMini",
  "command": ["/Users/fangjin/cred-manager.sh", "add", "NAME", "VALUE", "description"]
}
```
Then export:
```json
{
  "action": "run",
  "node": "MacMini",
  "command": ["/Users/fangjin/cred-manager.sh", "export"]
}
```

### Cross-Bot Sharing
- All 4 bots mount the same `env.secrets` file
- When one bot adds + exports a credential, all others can immediately `source` it
- No container restart needed for runtime access
- `env_file` credentials (loaded at start) require restart to refresh

## Backup

- **Automatic**: launchd job `com.openclaw.cred-backup` runs daily at 3:00 AM
- **Manual**: `~/cred-manager.sh backup`
- **Location**: `/Users/fangjin/nas/.credentials/`
- **Retention**: Last 30 backups + `store-latest.json`
- **Restore**: `~/cred-manager.sh restore` (uses latest) or `restore <specific-file>`

## Bot Skill

All 4 bots have `credential-manager` skill installed at `config/skills/credential-manager/SKILL.md`. The skill teaches bots to:
1. Check for existing credentials before asking user
2. Save new credentials via cred-manager.sh
3. Always export after add/delete
4. Never expose credential values in chat

## File Permissions

- `~/.credentials/` directory: 700 (owner only)
- `store.json`: 600 (owner read/write)
- `env.secrets`: 600 (owner read/write)
- Container mount: read-only (`:ro`)

## Troubleshooting

- **Bot can't read credentials**: Check mount exists: `docker exec <container> cat /mnt/credentials/env.secrets`
- **New credential not visible**: Did you run `export` after `add`?
- **NAS backup failing**: Check NAS mount: `mount | grep nas`
- **Restore needed**: `~/cred-manager.sh restore` then `export` then restart containers
