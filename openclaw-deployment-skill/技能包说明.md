## 🚀 快速使用

### 方式一：完整一键部署

```bash
cd D:\openclawVPS\openclaw-deployment-skill

./scripts/deploy.sh \
  --host 138.68.44.141 \
  --port 2222 \
  --key C:\Users\ennec\.ssh\id_ed25519 \
  --api-url https://ai.t8star.cn/v1/messages \
  --api-key sk-YOUR_KEY \
  --telegram-token YOUR_TELEGRAM_TOKEN \
  --discord-token YOUR_DISCORD_TOKEN
```

### 方式二：交互式部署

```bash
cd D:\openclawVPS\openclaw-deployment-skill
./scripts/deploy.sh
# 按提示输入配置信息
```

### 方式三：分步部署

```bash
# 1. 基础部署
./scripts/deploy.sh --host IP --api-url URL --api-key KEY

# 2. 配置消息频道（可选）
./scripts/setup-channels.sh --host IP --telegram-token TOKEN

# 3. 安装浏览器支持（可选）
./scripts/setup-browser.sh --host IP

# 4. 安装推荐技能（可选）
./scripts/install-skills.sh --host IP --all
```

## 🔧 技术特点

### 架构亮点

1. **双层 API 代理设计**
   ```
   OpenClaw → 本地代理(8022) → 自定义 API → Claude
   ```
   - 透明的模型名称转换
   - 不修改源代码
   - 易于维护和更新

2. **运行时修补机制**
   ```bash
   # 自动修补 node_modules 中的硬编码 URL
   sed -i 's|https://api.anthropic.com|http://127.0.0.1:8022|g'
   ```
   - 无需重新构建镜像
   - 启动时自动应用
   - 幂等操作

3. **Systemd 服务管理**
   - openclaw.service - 主服务
   - openclaw-proxy.service - API 代理服务
   - 自动重启策略
   - 依赖管理

4. **动态组件安装**
   - Chromium 按需安装
   - 不增加镜像体积
   - 快速部署

## 🎓 使用场景

### 场景 1：全新部署
```bash
./scripts/deploy.sh --host IP --api-url URL --api-key KEY --all
```
一键完成所有配置。

### 场景 2：仅更新 API 配置
```bash
./scripts/setup-api-proxy.sh --host IP --api-url NEW_URL --api-key NEW_KEY
```

### 场景 3：添加新频道
```bash
./scripts/setup-channels.sh --host IP --discord-token TOKEN
```

### 场景 4：备份配置
```bash
./scripts/backup.sh --host IP --output backup.tar.gz --include-workspace
```

### 场景 5：迁移到新服务器
```bash
# 在旧服务器
./scripts/backup.sh --host OLD_IP --output backup.tar.gz

# 在新服务器
./scripts/deploy.sh --host NEW_IP --api-url URL --api-key KEY
# 然后恢复配置
```

## ⚠️ 重要提示

### 安全建议

1. **修改默认 Token**
   ```yaml
   OPENCLAW_GATEWAY_TOKEN: <使用随机生成的 token>
   ```

2. **限制频道访问**
   ```json
   "allowFrom": ["你的用户ID"]
   ```

3. **启用防火墙**
   ```bash
   ufw allow 2222/tcp
   ufw enable
   ```

### 常见问题

1. **Connection Error**
   - 原因：API 代理服务停止
   - 解决：`systemctl restart openclaw-proxy`

2. **Discord 不响应**
   - 原因：未启用 Message Content Intent
   - 解决：在开发者门户的 Bot 页面启用

3. **浏览器超时**
   - 状态：预期行为
   - 说明：自动回退到 web_fetch

详细的故障排查请参考 `references/TROUBLESHOOTING.md`
