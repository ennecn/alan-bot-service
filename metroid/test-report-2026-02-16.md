## 事后排查与修复

### Bug 1：JSON Code Block 解析失败（根因）

**文件**: `src/engines/memory/encoder.ts`

LLM 返回的 JSON 被包裹在 ` ```json ... ``` ` 代码块中，而 `encodeAsync` 直接调用 `JSON.parse(raw)`，解析失败后被 `catch` 静默吞掉，所有编码结果全部丢弃。

**修复**: 新增 `parseJson()` 方法，先尝试直接解析，失败后提取代码块内容再解析：

```typescript
private parseJson(raw: string): any | null {
  try { return JSON.parse(raw); } catch { /* fall through */ }
  const match = raw.match(/```(?:json)?\s*([\s\S]*?)```/);
  if (match) {
    try { return JSON.parse(match[1].trim()); } catch { /* fall through */ }
  }
  return null;
}
```

### Bug 2：30% 采样 + 50% 二次采样 = 实际 15% 编码率

**文件**: `src/engines/memory/encoder.ts`, `src/engines/memory/index.ts`

旧设计在编码阶段做 30% 随机采样，Agent 回复还有额外 50% 二次采样，实际编码率仅 15%。5 轮测试中期望只有 0.75 条消息触发编码，几乎必然全部未命中。

**修复**:
- `maybeEncode` → `encode`：去掉 30% 采样，所有消息 100% 编码
- Agent 回复也 100% 编码（去掉 50% 二次采样）
- `config.ts` 中 `encodingSampleRate` 标记为 deprecated
- 设计理念转变：旧 = 编码时丢弃（"没记住"）→ 新 = 全部存储，召回时筛选（"想不起来"）

### 修复后测试状态
- 单元测试：119 passed, 2 failed（pre-existing forgetter NOT NULL bug，与本次修复无关）
- 待验证：重新跑多轮记忆测试确认 memories API 不再返回空数组
