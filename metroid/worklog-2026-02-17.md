| 文件 | 修改 |

# 2026-02-18 更新：Impulse Accumulator + 调试时钟

## 概要

重新设计主动消息触发机制。旧方案用简单情绪阈值判断，存在边界震荡、单维度、不够拟人三个问题。基于人类情感观察和学术调研，设计并实现了 **Impulse Accumulator（冲动累积器）** 模型，用连续冲动值替代离散触发条件。同时加入调试时钟，支持时间快进测试。

## 设计思想

四条人类情感观察：
1. 情绪本身不足以触发行为——需要"事件催化 + 情绪积累"的组合
2. 情绪会自然恢复，恢复速率因人格而异（乐天 vs 敏感）
3. 情绪是多维的（PAD 三轴），不能用单轴阈值判断
4. 个体敏感度不同，同样的刺激对不同人格产生不同冲动

核心模型：
```
信号源（情绪模式 + 空闲 + 时间窗口）
    ↓ 加权求和 × 事件门控
  impulse 累积器（0~1）
    ↓ 超过动态阈值？
  克制机制（概率性触发）
    ↓ 通过？
  生成主动消息
```

Impulse 数学模型：
- **Gain**: `Σ(signal.weight × activation × event_gate) × dt_hours`
- **Decay**: `decayRate × dt_hours × (1 + restraint)`
- **Event gate**: emotion_pattern 信号需要活跃事件才能贡献增益
- **触发**: `sigmoid(10 × expressiveness × (impulse - dynamicThreshold))`
- **Suppression bonus**: 被压制越多次阈值越低，保证最终触发

# 2026-02-18 更新：情感引擎 + 成长引擎 LLM 语义分析

## 概要

将情感引擎和成长引擎从纯规则驱动升级为 LLM 语义分析优先、规则兜底的双模式架构。LLM 调用不阻塞用户响应（在 `onResponse()` 中异步执行），失败时无缝降级到原有关键词/规则方法。

## 设计原则

1. **LLM 优先，规则兜底**：LLM 调用失败时无缝降级到现有关键词/规则方法
2. **零延迟影响**：`onResponse()` 在用户收到回复后执行，LLM 分析不阻塞用户
3. **自然限流**：情感引擎已有 30s 间隔限制，成长引擎每 10 条消息才评估一次
4. **复用现有模式**：参照 `graph-rag.ts` 的 OpenAI-compatible fetch 调用模式

## 代码修改

| | 新增 `parsePADJson()` — 解析 LLM 返回的 JSON，clamp 到安全范围 |
| | `onResponse()` 改为 LLM 优先 + 关键词 fallback |

LLM Prompt 核心：分析对话片段的情感变化，返回 PAD delta JSON（pleasure ±0.3, arousal ±0.3, dominance ±0.2）。拼接用户消息 + 最近 2 条历史 + AI 回复，截断 800 字符。

# 2026-02-18 更新：Web-IM Adapter（WebSocket 实时聊天）

## 概要

在现有 HTTP REST API 基础上增加 WebSocket 双向通信能力，实现实时聊天和主动消息推送。手写 RFC 6455 协议实现，零新依赖。

## 设计原则

1. **增量升级**：WebSocket 与 HTTP API 共存，不替换现有接口
2. **零依赖**：用 Node.js 原生 `crypto` + `net` 手写 WebSocket 协议
3. **推送优先**：主动消息通过 WS 实时推送，不再需要轮询
4. **协议简洁**：JSON 消息 + type 字段
